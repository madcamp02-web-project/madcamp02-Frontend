# 🤖 AI Prompting Log - Stock-Persona 프로젝트

> 대화 일시: 2026-01-16  
> 목적: 프로젝트 명세서 작성 및 개발 계획 수립

---

## 📝 대화 흐름

### 1️⃣ 초기 분석 요청
**프롬프트 방식:** 컨텍스트 주입 + 분석 지시

```
[사용자]
- 프로젝트 명세서(Ver 1.0) 텍스트 제공
- @app @components @lib 등 파일들 @ 멘션으로 첨부
- "분석해서 보고서와 계획서를 마크다운으로 출력해줘"
```

**결과:** GAP 분석 보고서 Ver 1.1 (현황 분석 + 누락 기능 목록)

---

### 2️⃣ 상세 명세서 기반 계획서 생성
**프롬프트 방식:** 파일 첨부 + 다중 태스크 + 제약 조건

```
[사용자]
- @docs/FULL_SPECIFICATION.md 첨부 (Ver 2.0)
- "오류 없는지 확인하고"
- "프론트/백엔드 계획서 2개 만들어"
- "자세한 코드는 작성하지 말고 파일 설계방법 정도로"
```

**결과:** 
- `docs/FRONTEND_DEVELOPMENT_PLAN.md` 생성
- `docs/BACKEND_DEVELOPMENT_PLAN.md` 생성

---

### 3️⃣ 명세서 수정 요청
**프롬프트 방식:** 특정 부분 지적 + 수정 지시

```
[사용자]
"프론트엔드 구조 ⚠️ 미세 수정 필요 현재 라우트 구조 반영"
"이거 수정해서 해당 마크다운에 반영해줄래?"
```

**결과:** `FULL_SPECIFICATION.md` 6.2 디렉토리 구조 → 플랫 구조로 수정

---

### 4️⃣ 문서 간 일치성 검증 요청
**프롬프트 방식:** 다중 파일 비교 + 확인 요청

```
[사용자]
@docs/BACKEND_DEVELOPMENT_PLAN.md
@docs/FRONTEND_DEVELOPMENT_PLAN.md
@docs/FULL_SPECIFICATION.md
"전반적으로 서로 잘 맞는지 확인해봐"
```

**결과:** 불일치 1건 발견 (라우트 그룹 구조)

---

### 5️⃣ 불일치 수정 확인
**프롬프트 방식:** 예시 제시 + 확인 질문

```
[사용자]
"FRONTEND_DEV_PLAN에서 (auth)와 (main) 라우트 그룹 사용 중"
"FULL_SPECIFICATION은 플랫 구조"
"이거 고쳤음?"
```

**결과:** `FRONTEND_DEVELOPMENT_PLAN.md` 4곳 수정
- 3.1 디렉토리 구조
- 4.1 페이지 라우팅 맵
- 9.2 인증 미들웨어
- Phase 2 로드맵

---

### 6️⃣ 최종 엄격 검증 요청
**프롬프트 방식:** 강조 표현 + 요약 파일 지정

```
[사용자]
"서로 하나하나 엄격하고 꼼꼼하게 확인해보고"
"아주 간략하게 @요약.md 에 짧게 정리해"
```

**결과:** 
- 9개 항목 교차 검증 완료 (모두 일치)
- `요약.md` 업데이트

---

### 7️⃣ 대화 로그 요청
**프롬프트 방식:** 메타 요청 + 간결화 제약

```
[사용자]
"전체 대화를 ai_log.md로 정리해"
"너무 길게 하지 말고 짧고 간략하게"
```

**결과:** 현재 파일

## 💡 사용된 프롬프팅 기법

| 기법 | 설명 | 예시 |
|------|------|------|
| **컨텍스트 주입** | `@` 멘션으로 파일 첨부 | `@docs/FULL_SPECIFICATION.md` |
| **다중 태스크** | 한 번에 여러 작업 요청 | "확인하고 + 2개 만들어" |
| **제약 조건** | 출력 형식/범위 제한 | "코드 말고 설계만", "짧게" |
| **특정 지적** | 문제 부분 직접 명시 | "이 부분 수정해줘" |
| **확인 질문** | 작업 완료 여부 확인 | "이거 고쳤음?" |
| **강조 표현** | 중요도 전달 | "엄격하고 꼼꼼하게" |
